## Prerequisites

Before you begin, ensure you have the following installed:

- **Java 25** - Required for running the Spring Boot backend
- **Maven** - For building the Java project
- **Node.js and npm** - For running the Angular frontend
- **Docker and Docker Compose** - For running PostgreSQL and AiStore
- **Doppler CLI** - For managing environment variables securely
- **Just** (optional) - For using the convenience commands in the justfile

<Warning>
Make sure you have a Doppler account set up and are logged in via the CLI. Brautcloud uses Doppler for secure environment variable management.
</Warning>

## Clone the repositories

<Steps>
  <Step title="Clone the backend repository">
    ```bash
    git clone https://github.com/domenicwalther/brautcloud-backend.git
    cd brautcloud-backend
    ```
  </Step>
  
  <Step title="Clone the frontend repository">
    ```bash
    cd ..
    git clone https://github.com/domenicwalther/brautcloud-frontend.git
    ```
  </Step>
</Steps>

## Set up environment variables

Brautcloud requires several environment variables for database and S3 storage configuration.

<Steps>
  <Step title="Configure Doppler">
    Add the following variables to your Doppler project/config:

    | Variable | Description | Example |
    |----------|-------------|----------|
    | `AWS_ACCESS_KEY_ID` | Access key for local S3 (AiStore) | `minioadmin` |
    | `AWS_SECRET_ACCESS_KEY` | Secret key for local S3 (AiStore) | `minioadmin` |
    | `AWS_ENDPOINT` | Endpoint URL for the local S3 server | `http://localhost:9000` |
    | `POSTGRES_URL` | JDBC URL for PostgreSQL database | `jdbc:postgresql://localhost:5432/mydatabase` |
    | `POSTGRES_USER` | PostgreSQL username | `myuser` |
    | `POSTGRES_PW` | PostgreSQL password | `secret` |
    | `JWT_SECRET` | Secret key for JWT token signing | `your-secret-key-here` |
  </Step>
  
  <Step title="Verify Doppler access">
    ```bash
    doppler --help
    ```
    
    Make sure you're logged in and have access to your Doppler project.
  </Step>
</Steps>

<Note>
The values in the table match the default configuration in `compose.yaml`. You can adjust them as needed for your environment.
</Note>

## Start the infrastructure services

Brautcloud requires PostgreSQL for data storage and AiStore for photo storage.

<Steps>
  <Step title="Navigate to the backend directory">
    ```bash
    cd brautcloud-backend
    ```
  </Step>
  
  <Step title="Start Docker Compose">
    ```bash
    docker compose up -d
    ```
    
    This starts two services:
    - **PostgreSQL** on port 5432
    - **AiStore** on ports 9000 (API) and 9001 (Console)
  </Step>
  
  <Step title="Verify services are running">
    ```bash
    docker compose ps
    ```
    
    You should see both `postgres` and `aistor` services in the "running" state.
  </Step>
</Steps>

<Accordion title="Setting up AiStore license">
  AiStore requires a license file to run. You'll need to:
  
  1. Obtain a MinIO AiStore license
  2. Save it to `~/minio/minio.license`
  3. Ensure the volume mount in `compose.yaml` points to the correct location
  
  The default configuration expects the license at `${HOME}/minio/minio.license`.
</Accordion>

## Start the backend

<Steps>
  <Step title="Build the backend">
    ```bash
    cd brautcloud-backend
    mvn clean install
    ```
  </Step>
  
  <Step title="Run with Doppler">
    The backend includes a Spring run configuration that automatically starts with Doppler:
    
    ```bash
    doppler run -- mvn spring-boot:run
    ```
    
    The backend will start on the default Spring Boot port (typically 8080).
  </Step>
  
  <Step title="Verify the backend is running">
    ```bash
    curl http://localhost:8080/actuator/health
    ```
    
    You should receive a response indicating the application is up.
  </Step>
</Steps>

<Warning>
At this stage, the Docker Compose Spring integration is disabled. You must run `docker compose up` manually before starting the Spring Boot application.
</Warning>

## Start the frontend

<Steps>
  <Step title="Install dependencies">
    ```bash
    cd brautcloud-frontend
    npm install
    ```
  </Step>
  
  <Step title="Start the development server">
    ```bash
    npm start
    ```
    
    Or using Angular CLI directly:
    
    ```bash
    ng serve
    ```
    
    The frontend will start on `http://localhost:4200`.
  </Step>
  
  <Step title="Open in browser">
    Navigate to `http://localhost:4200` in your web browser to see the Brautcloud interface.
  </Step>
</Steps>

## Using Just for convenience

If you have [Just](https://github.com/casey/just) installed, you can use the convenience commands in the root justfile:

<CodeGroup>
```bash Start both services
just dev
```

```bash Backend only
just backend dev
```

```bash Frontend only
just frontend dev
```
</CodeGroup>

The `just dev` command starts both the backend and frontend in parallel.

## Create your first event

Once both the backend and frontend are running:

<Steps>
  <Step title="Register an account">
    Create a new user account through the registration flow in the web interface.
  </Step>
  
  <Step title="Create an event">
    After logging in, create your first wedding event. Give it a name and date.
  </Step>
  
  <Step title="Upload photos">
    Start uploading photos to your event. The photos will be stored in AiStore and metadata in PostgreSQL.
  </Step>
</Steps>

<Info>
You can access the AiStore console at `http://localhost:9001` to view uploaded files and manage buckets.
</Info>

## Troubleshooting

<Accordion title="Backend fails to connect to database">
  - Verify PostgreSQL is running: `docker compose ps`
  - Check your Doppler environment variables match the values in `compose.yaml`
  - Ensure the `POSTGRES_URL` includes the correct hostname (usually `localhost`)
</Accordion>

<Accordion title="Photos fail to upload">
  - Verify AiStore is running on port 9000
  - Check your `AWS_ENDPOINT` environment variable points to `http://localhost:9000`
  - Ensure the `brautcloud` bucket exists in AiStore
  - Verify your AiStore license is valid and mounted correctly
</Accordion>

<Accordion title="Frontend can't connect to backend">
  - Verify the backend is running and accessible at `http://localhost:8080`
  - Check for CORS configuration issues in the Spring Boot application
  - Look at the browser console for specific error messages
</Accordion>
