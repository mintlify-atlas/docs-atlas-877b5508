## System overview

Brautcloud is built as a modern three-tier web application with a clear separation between frontend, backend, and data storage layers.

```
┌─────────────────────────────────────────────────┐
│              Angular Frontend                   │
│           (Port 4200)                          │
│  - Angular 21                                  │
│  - Tailwind CSS                                │
│  - RxJS                                        │
└─────────────────┬───────────────────────────────┘
                  │ HTTP/REST API
                  │ JWT Authentication
                  │
┌─────────────────▼───────────────────────────────┐
│         Spring Boot Backend                     │
│           (Port 8080)                          │
│  - Java 25                                     │
│  - Spring Boot 4.0.2                           │
│  - Spring Security + JWT                       │
│  - Spring Data JPA                             │
└─────────┬───────────────────┬───────────────────┘
          │                   │
          │ JDBC              │ AWS S3 SDK
          │                   │
┌─────────▼─────────┐  ┌──────▼──────────────────┐
│   PostgreSQL      │  │   AiStore (MinIO)       │
│   (Port 5432)     │  │   (Ports 9000/9001)     │
│                   │  │                         │
│ - User data       │  │ - Photo storage         │
│ - Event metadata  │  │ - S3-compatible API     │
│ - Relationships   │  │ - Object storage        │
└───────────────────┘  └─────────────────────────┘
```

#### Security layer

<Info>
Brautcloud uses **JWT (JSON Web Tokens)** for stateless authentication:

- Tokens expire after 24 hours (86400000 ms)
- Refresh tokens last 30 days
- Secret key managed via Doppler environment variables
- Spring Security integration for endpoint protection
</Info>

Configuration from `application.properties:22-27`:

```properties
jwt.token.secret=${JWT_SECRET}
jwt.token.expires=86400000
jwt.token.refreshExpires=30d
```

#### File upload handling

The backend supports photo uploads up to 10MB:

```properties
spring.servlet.multipart.max-file-size=10MB
spring.servlet.multipart.max-request-size=10MB
```

## Data storage

### PostgreSQL database

Brautcloud uses **PostgreSQL** for relational data storage.

#### Configuration

```properties
spring.sql.init.mode=always
spring.datasource.url=${POSTGRES_URL}
spring.datasource.username=${POSTGRES_USER}
spring.datasource.password=${POSTGRES_PW}
```

#### Data stored

- User accounts and authentication credentials
- Wedding event metadata (name, date, organizer)
- Photo metadata (filename, upload date, user)
- Relationships between users, events, and photos

<Note>
The database schema is initialized automatically on startup using JPA entity definitions. Schema creation is managed by Hibernate.
</Note>

### AiStore object storage

**AiStore** (powered by MinIO) provides S3-compatible object storage for photos.

#### Configuration

From `application.properties:14-19`:

```properties
aws.accessKeyId=${AWS_ACCESS_KEY_ID}
aws.secretAccessKey=${AWS_SECRET_ACCESS_KEY}
aws.s3.endpoint=${AWS_ENDPOINT}
aws.s3.region=us-east-1
aws.s3.bucket=brautcloud
```

#### Why AiStore?

- **S3-compatible API** - Works with AWS S3 SDK without modification
- **Self-hosted** - Run locally or on-premise for data sovereignty
- **Scalable** - Designed for large-scale object storage
- **Cost-effective** - No cloud storage costs for local/on-premise deployments

#### Storage workflow

<Steps>
  <Step title="Photo upload request">
    User uploads a photo through the Angular frontend
  </Step>
  
  <Step title="Backend validation">
    Spring Boot validates file size (max 10MB) and type
  </Step>
  
  <Step title="S3 upload">
    Backend uploads photo to AiStore using AWS S3 SDK
  </Step>
  
  <Step title="Metadata storage">
    Photo metadata is saved to PostgreSQL with reference to S3 object key
  </Step>
  
  <Step title="Response">
    Frontend receives confirmation and displays the photo
  </Step>
</Steps>

## API communication

### REST endpoints

The frontend communicates with the backend via RESTful HTTP endpoints:

- **Authentication** - POST `/api/auth/login`, `/api/auth/register`
- **Events** - CRUD operations on wedding events
- **Photos** - Upload, retrieve, and delete photos
- **Users** - User profile management

### Authentication flow

<Steps>
  <Step title="User login">
    User submits credentials to `/api/auth/login`
  </Step>
  
  <Step title="JWT issuance">
    Backend validates credentials and issues JWT access token + refresh token
  </Step>
  
  <Step title="Token storage">
    Frontend stores tokens (typically in memory or httpOnly cookies)
  </Step>
  
  <Step title="Authenticated requests">
    Frontend includes JWT in `Authorization: Bearer <token>` header
  </Step>
  
  <Step title="Backend validation">
    Spring Security validates JWT on each request
  </Step>
  
  <Step title="Token refresh">
    When access token expires, use refresh token to get new access token
  </Step>
</Steps>
