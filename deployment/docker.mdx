## Overview

Brautcloud can be deployed using Docker containers. This guide covers building Docker images and running the application using Docker Compose.

## Prerequisites

- Docker Engine 20.10 or higher
- Docker Compose V2 or higher
- 4GB+ RAM available for containers
- 20GB+ disk space

## Project Structure

The `compose.yaml` file in the backend repository defines the infrastructure services:

```yaml:brautcloud-backend/compose.yaml
services:
  postgres:
    image: 'postgres:latest'
    environment:
      - 'POSTGRES_DB=mydatabase'
      - 'POSTGRES_PASSWORD=secret'
      - 'POSTGRES_USER=myuser'
    ports:
      - '5432:5432'
  aistor:
    image: quay.io/minio/aistor/minio:latest
    command: minio server /mnt/data --license /minio.license
    ports:
      - '9000:9000'
      - '9001:9001'
    volumes:
      - /${HOME}/minio/certs:/etc/minio/certs
      - /${HOME}/minio/data:/mnt/data
      - /${HOME}/minio/minio.license:/minio.license
    tty: true
```

<Note>
The current `compose.yaml` only includes infrastructure services (PostgreSQL and AiStor). You'll need to extend it to include the backend and frontend services.
</Note>

## Quick Start with Docker Compose

### 1. Start Infrastructure Services

The existing compose file provides PostgreSQL and AiStor (MinIO):

```bash
cd brautcloud-backend
docker compose up -d
```

This starts:
- **PostgreSQL**: Available at `localhost:5432`
- **AiStor/MinIO**: 
  - API at `localhost:9000`
  - Console at `localhost:9001`

### 2. Configure AiStor/MinIO

Create the required S3 bucket:

```bash
# Install MinIO Client
brew install minio/stable/mc  # macOS
# or
wget https://dl.min.io/client/mc/release/linux-amd64/mc

# Configure MinIO client
mc alias set myminio http://localhost:9000 minioadmin minioadmin

# Create bucket
mc mb myminio/brautcloud

# Set bucket policy (public read for uploaded images)
mc anonymous set download myminio/brautcloud
```

Or use the MinIO Console at http://localhost:9001 (login: minioadmin/minioadmin).

### 3. Set Environment Variables

Create a `.env` file for the backend (see [Environment Variables](/deployment/environment-variables)):

```bash
# .env file (DO NOT commit to git)
POSTGRES_URL=postgres://localhost:5432/mydatabase
POSTGRES_USER=myuser
POSTGRES_PW=secret

AWS_ACCESS_KEY_ID=minioadmin
AWS_SECRET_ACCESS_KEY=minioadmin
AWS_ENDPOINT=http://localhost:9000

JWT_SECRET=your-generated-secret-min-32-chars-here
```

### 4. Run Backend and Frontend

**Backend (with Doppler):**
```bash
cd brautcloud-backend
doppler run -- mvn spring-boot:run
```

**Backend (with .env file):**
```bash
cd brautcloud-backend
export $(cat .env | xargs)
mvn spring-boot:run
```

**Frontend:**
```bash
cd brautcloud-frontend
npm install
npm start
```

## Building Docker Images

### Backend Docker Image

Create `Dockerfile` in `brautcloud-backend/`:

```dockerfile
# Multi-stage build for Spring Boot
FROM maven:3.9-eclipse-temurin-25 AS build
WORKDIR /app

# Copy pom.xml and download dependencies
COPY pom.xml .
RUN mvn dependency:go-offline -B

# Copy source code and build
COPY src ./src
RUN mvn clean package -DskipTests

# Runtime stage
FROM eclipse-temurin:25-jre
WORKDIR /app

# Copy jar from build stage
COPY --from=build /app/target/*.jar app.jar

# Expose port
EXPOSE 8080

# Run application
ENTRYPOINT ["java", "-jar", "app.jar"]
```

**Build the image:**
```bash
cd brautcloud-backend
docker build -t brautcloud-backend:latest .
```

### Frontend Docker Image

Create `Dockerfile` in `brautcloud-frontend/`:

```dockerfile
# Build stage
FROM node:18-alpine AS build
WORKDIR /app

# Copy package files
COPY package*.json ./
RUN npm ci

# Copy source and build
COPY . .
RUN npm run build

# Runtime stage with nginx
FROM nginx:alpine

# Copy nginx configuration
COPY nginx.conf /etc/nginx/nginx.conf

# Copy built app
COPY --from=build /app/dist/brautcloud-frontend/browser /usr/share/nginx/html

EXPOSE 80

CMD ["nginx", "-g", "daemon off;"]
```

**Create `nginx.conf` in `brautcloud-frontend/`:**

```nginx
events {
    worker_connections 1024;
}

http {
    include /etc/nginx/mime.types;
    default_type application/octet-stream;
    
    server {
        listen 80;
        server_name localhost;
        root /usr/share/nginx/html;
        index index.html;
        
        # Angular routing
        location / {
            try_files $uri $uri/ /index.html;
        }
        
        # API proxy (optional)
        location /api/ {
            proxy_pass http://backend:8080/api/;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
        }
    }
}
```

**Build the image:**
```bash
cd brautcloud-frontend
docker build -t brautcloud-frontend:latest .
```

## Complete Docker Compose Setup

Create an extended `docker-compose.prod.yaml` that includes all services:

```yaml
services:
  # Database
  postgres:
    image: postgres:latest
    environment:
      - POSTGRES_DB=brautcloud
      - POSTGRES_USER=brautcloud
      - POSTGRES_PASSWORD=${POSTGRES_PASSWORD}
    volumes:
      - postgres_data:/var/lib/postgresql/data
    ports:
      - "5432:5432"
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U brautcloud"]
      interval: 10s
      timeout: 5s
      retries: 5
    networks:
      - brautcloud-network

  # S3-compatible storage
  aistor:
    image: quay.io/minio/aistor/minio:latest
    command: minio server /data --console-address ":9001"
    environment:
      - MINIO_ROOT_USER=${MINIO_ROOT_USER:-minioadmin}
      - MINIO_ROOT_PASSWORD=${MINIO_ROOT_PASSWORD:-minioadmin}
    volumes:
      - aistor_data:/data
    ports:
      - "9000:9000"
      - "9001:9001"
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:9000/minio/health/live"]
      interval: 30s
      timeout: 20s
      retries: 3
    networks:
      - brautcloud-network

  # Backend
  backend:
    image: brautcloud-backend:latest
    build:
      context: ./brautcloud-backend
      dockerfile: Dockerfile
    environment:
      - POSTGRES_URL=postgres://postgres:5432/brautcloud
      - POSTGRES_USER=brautcloud
      - POSTGRES_PW=${POSTGRES_PASSWORD}
      - AWS_ACCESS_KEY_ID=${MINIO_ROOT_USER:-minioadmin}
      - AWS_SECRET_ACCESS_KEY=${MINIO_ROOT_PASSWORD:-minioadmin}
      - AWS_ENDPOINT=http://aistor:9000
      - JWT_SECRET=${JWT_SECRET}
      - SPRING_PROFILES_ACTIVE=prod
    depends_on:
      postgres:
        condition: service_healthy
      aistor:
        condition: service_healthy
    ports:
      - "8080:8080"
    networks:
      - brautcloud-network
    restart: unless-stopped

  # Frontend
  frontend:
    image: brautcloud-frontend:latest
    build:
      context: ./brautcloud-frontend
      dockerfile: Dockerfile
    depends_on:
      - backend
    ports:
      - "80:80"
    networks:
      - brautcloud-network
    restart: unless-stopped

volumes:
  postgres_data:
  aistor_data:

networks:
  brautcloud-network:
    driver: bridge
```

### Using the Complete Setup

**1. Create `.env` file:**
```bash
# .env (DO NOT commit)
POSTGRES_PASSWORD=your_secure_password
MINIO_ROOT_USER=admin
MINIO_ROOT_PASSWORD=your_minio_password
JWT_SECRET=your_jwt_secret_min_32_chars
```

**2. Build and start all services:**
```bash
docker compose -f docker-compose.prod.yaml up -d --build
```

**3. Initialize S3 bucket:**
```bash
# Wait for services to be healthy
docker compose -f docker-compose.prod.yaml ps

# Create bucket
docker compose -f docker-compose.prod.yaml exec aistor \
  mc mb /data/brautcloud
```

**4. Access the application:**
- Frontend: http://localhost
- Backend API: http://localhost:8080
- MinIO Console: http://localhost:9001

## Production Deployment

### Using Docker Swarm

**1. Initialize swarm:**
```bash
docker swarm init
```

**2. Create secrets:**
```bash
echo "your_secure_password" | docker secret create postgres_password -
echo "your_jwt_secret" | docker secret create jwt_secret -
echo "your_minio_password" | docker secret create minio_password -
```

**3. Deploy stack:**
```bash
docker stack deploy -c docker-compose.prod.yaml brautcloud
```

### Using Kubernetes

For Kubernetes deployment:

1. Create Kubernetes manifests (Deployments, Services, ConfigMaps, Secrets)
2. Use Helm charts for easier management
3. Consider managed services (RDS for PostgreSQL, S3 for storage)

**Example Kubernetes secret:**
```bash
kubectl create secret generic brautcloud-secrets \
  --from-literal=postgres-password=your_password \
  --from-literal=jwt-secret=your_jwt_secret \
  --from-literal=minio-password=your_minio_password
```

## Monitoring and Maintenance

### View Logs

```bash
# All services
docker compose logs -f

# Specific service
docker compose logs -f backend

# Last 100 lines
docker compose logs --tail=100 backend
```

### Health Checks

```bash
# Check service status
docker compose ps

# Check backend health
curl http://localhost:8080/actuator/health
```

### Database Backups

```bash
# Backup database
docker compose exec postgres pg_dump -U brautcloud brautcloud > backup.sql

# Restore database
docker compose exec -T postgres psql -U brautcloud brautcloud < backup.sql
```

### S3 Storage Backups

```bash
# Backup S3 data
mc mirror myminio/brautcloud ./backup/brautcloud

# Restore S3 data
mc mirror ./backup/brautcloud myminio/brautcloud
```

### Update Application

```bash
# Pull latest code
git pull

# Rebuild and restart
docker compose -f docker-compose.prod.yaml up -d --build

# Or for specific service
docker compose -f docker-compose.prod.yaml up -d --build backend
```

## Performance Optimization

### Resource Limits

Add resource constraints to `docker-compose.prod.yaml`:

```yaml
services:
  backend:
    deploy:
      resources:
        limits:
          cpus: '2'
          memory: 2G
        reservations:
          cpus: '1'
          memory: 1G
```

### Volume Performance

For better I/O performance on Linux:

```yaml
volumes:
  postgres_data:
    driver: local
    driver_opts:
      type: none
      o: bind
      device: /mnt/data/postgres
```

## Troubleshooting

### Backend Won't Start

```bash
# Check logs
docker compose logs backend

# Common issues:
# 1. Database not ready - wait for health check
# 2. Missing environment variables - check .env file
# 3. Port already in use - change port mapping
```

### Database Connection Issues

```bash
# Test database connection
docker compose exec backend sh -c '
  apt-get update && apt-get install -y postgresql-client
  psql $POSTGRES_URL -c "SELECT version();"
'
```

### S3 Connection Issues

```bash
# Test S3 connection from backend
docker compose exec backend sh -c '
  curl -v $AWS_ENDPOINT
'

# Check bucket exists
mc ls myminio/brautcloud
```

### Frontend Not Loading

```bash
# Check nginx logs
docker compose logs frontend

# Test backend API connection
curl http://localhost:8080/actuator/health
```

## Security Best Practices

<Warning>
Always use strong passwords and secure secrets in production.
</Warning>

1. **Use secrets management** - Docker secrets or external secret managers
2. **Run as non-root user** - Add `USER` directive in Dockerfiles
3. **Scan images** - Use `docker scan` or Trivy for vulnerabilities
4. **Use private registry** - Don't expose images publicly
5. **Enable TLS** - Use reverse proxy (nginx/Traefik) with SSL certificates
6. **Network isolation** - Use internal networks for service communication
7. **Regular updates** - Keep base images and dependencies updated
