## Overview

Retrieves the currently authenticated user's profile information including their personal details, email, and associated events.

## Authentication

This endpoint requires JWT authentication. Include the access token in the Authorization header.

## Request

<ParamField header="Authorization" type="string" required>
  Bearer token obtained from the `/api/auth/login` endpoint
</ParamField>

## Response

<ResponseField name="id" type="long">
  Unique identifier for the user
</ResponseField>

<ResponseField name="lastName" type="string">
  Last name (surname) shared by the couple
</ResponseField>

<ResponseField name="firstNameCoupleOne" type="string">
  First name of the first person in the couple
</ResponseField>

<ResponseField name="firstNameCoupleTwo" type="string">
  First name of the second person in the couple
</ResponseField>

<ResponseField name="createdAt" type="datetime">
  Timestamp when the user account was created (ISO 8601 format)
</ResponseField>

<ResponseField name="email" type="string">
  User's email address (unique)
</ResponseField>

<ResponseField name="emailVerified" type="boolean">
  Whether the user's email address has been verified
</ResponseField>

<ResponseField name="events" type="array">
  List of events associated with this user
  
  <Expandable title="Event object">
    <ResponseField name="id" type="long">
      Event identifier
    </ResponseField>
    
    <ResponseField name="name" type="string">
      Event name
    </ResponseField>
    
    <ResponseField name="date" type="datetime">
      Event date
    </ResponseField>
  </Expandable>
</ResponseField>

### Security

This endpoint is protected by JWT authentication. The token must be:
- Valid and not expired
- Issued by the Brautcloud authentication service
- Include the user's email in the token claims

### Rate Limiting

Standard rate limiting applies: 100 requests per minute per user.
